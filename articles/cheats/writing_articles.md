# 📝 Полный гайд: Создание контента для VR Kvant

Этот документ поможет вам быстро освоить формат оформления уроков и шпаргалок для нашего портала. Мы используем **Markdown** для текста и **JSON** для управления структурой сайта.

---

## 1. Основы оформления (Markdown)

Markdown — это текстовый стандарт, который превращается в красивую верстку на сайте.

### Заголовки
Для создания структуры используйте решетки (`#`). Чем больше решеток, тем меньше заголовок.

**Как писать:**
```markdown
# Название статьи (Используется только ОДИН раз)
## Заголовок раздела
### Подзаголовок
```

### Списки
Нумерованные и маркированные списки создаются очень просто.

**Как писать:**
```markdown
1. Включите шлем Meta Quest.
2. Подключите кабель к ПК.
* Убедитесь, что горит зеленый индикатор.
* Не забудьте надеть ремешки!
```

**Как это выглядит:**
1. Включите шлем Meta Quest.
2. Подключите кабель к ПК.
* Убедитесь, что горит зеленый индикатор.
* Не забудьте надеть ремешки!

### Акценты в тексте
Используйте выделение, чтобы акцентировать внимание на важных кнопках или терминах.

**Как писать:**
```markdown
Нажмите кнопку **Play**, чтобы запустить сцену. *Первый запуск может занять пару секунд.*
```

**Как это выглядит:**
Нажмите кнопку **Play**, чтобы запустить сцену. *Первый запуск может занять пару секунд.*

## 🎨 Умные цветные цитаты

Наш сайт автоматически распознает тип вашей подсказки и красит ее в нужный цвет! Просто начните цитату со специального слова:

> **Важно:** Блок станет красным/оранжевым. Используйте для критических ошибок или моментов, где можно сломать проект. *(Также работают слова: Внимание:, Предупреждение:)*

> **Совет:** Блок станет зеленым. Отлично подходит для описания горячих клавиш или хитростей, ускоряющих работу. *(Также работает слово: Лайфхак:)*

> **Заметка:** Блок станет синим. Используйте для дополнительных фактов и теоретических справок. *(Также работает слово: Информация:)*

> Если начать просто с любого другого текста без этих слов, цитата будет стандартного серо-фиолетового цвета.

> **Совет:** Всегда оставляйте пустую строку до и после списков, цитат и блоков кода, чтобы они корректно отображались.

## 2. Вставка медиафайлов

### Изображения
Для вставки изображения используйте квадратные и круглые скобки.

**Как писать:**
```markdown
![Пример работы нодов в Unreal](https://via.placeholder.com/600x150.png?text=Скриншот+вашего+кода)
```

**Как это выглядит:**
![Пример работы нодов в Unreal](https://via.placeholder.com/600x150.png?text=Скриншот+вашего+кода)

> **Важно:** В реальных статьях вы должны ссылаться на локальные файлы в папке `img`, например: `![Окно](img/unreal/interface_1.png)`. Названия файлов — только латиница!

### Код (Скрипты)
Для вставки программного кода (C#, C++, Python) используйте три обратных апострофа и название языка. 

**Как писать:**
<pre>
```csharp
public class PlayerController : MonoBehaviour {
    void Update() {
        Debug.Log("Hello VR!");
    }
}
```
</pre>

**Как это выглядит:**
```csharp
public class PlayerController : MonoBehaviour {
    void Update() {
        Debug.Log("Hello VR!");
    }
}
```

## 4. Как опубликовать материал на сайте

Создание файла — это 50% работы. Чтобы сайт "увидел" статью, её нужно прописать в манифесте.

### Добавление Урока (в раздел Треки)
1. Откройте `articles/tracks.json`.
2. Найдите массив `lessons` внутри нужного трека.
3. Добавьте объект (не забывайте запятую перед ним, если он не первый):

```json
{ "title": "Название вашего урока", "file": "название_файла.md" }
```

### Добавление Шпаргалки
1. Откройте `articles/cheats.json`.
2. Добавьте новую запись в конец списка:

```json
{ "title": "Название шпаргалки", "file": "название_файла.md" }
```

## 💡 Чек-лист проверки (Перед сохранением)

* **Названия**: Файлы и папки названы латиницей (без пробелов, через `_`).
* **Пути**: Вы проверили пути до картинок.
* **Закрытие тегов**: Все блоки кода и жирный текст закрыты.
* **JSON**: Манифест-файлы сохранены корректно, нет лишних запятых перед закрывающей скобкой `]`.
