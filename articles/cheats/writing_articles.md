# 📝 Полный гайд для педагогов: Создание контента для VR Kvant

Этот документ поможет вам быстро освоить формат оформления уроков и шпаргалок для нашего портала. Мы используем **Markdown** для текста и **JSON** для управления структурой сайта.

---

## 1. Основы оформления (Markdown)

Markdown — это текстовый стандарт, который превращается в красивую верстку на сайте.

### Заголовки
* `# Название статьи` — Используется только ОДИН раз в самом начале файла.
* `## Заголовок раздела` — Для крупных блоков.
* `### Подзаголовок` — Для уточнения внутри разделов.

### Списки
* Используйте `*` или `-` для маркированных списков.
* Используйте `1.`, `2.`, `3.` для нумерованных инструкций.

### Акценты
* `**Жирный текст**` превратится в **Жирный текст** — используйте для терминов.
* `*Курсив*` превратится в *Курсив* — для примечаний.
* `> Цитата` — используется для "Лайфхаков" или "Важных предупреждений".

## 2. Вставка медиафайлов

### Изображения
Для вставки изображения используйте конструкцию:
`![Описание](img/папка_трека/название_файла.png)`

**Правила для картинок:**
1. Загружайте файлы в `img/blender/`, `img/unity/` или `img/unreal/`.
2. Названия файлов — только латиница (например, `screenshot_01.png`).

> [Место для картинки: Сделайте скриншот структуры папок в репозитории, где выделена папка img и соответствующие подпапки, и вставьте ссылку на него здесь]

### Код (Скрипты)
Для вставки программного кода (например, C# для Unity) используйте три обратных апострофа (\`\`\`) и название языка (например, csharp). 

Пример того, как это нужно писать в файле:

```csharp
public class PlayerController : MonoBehaviour {
    void Update() {
        // Логика движения здесь
    }
}
```

### Видео (YouTube)
Наш сайт поддерживает вставку через `iframe`. На YouTube нажмите «Поделиться» -> «Встроить» и скопируйте код. Вставьте его прямо в `.md` файл.

> [Место для картинки: Сделайте скриншот окна YouTube с открытой вкладкой "Встроить", где выделен фрагмент кода iframe]

## 3. Файлы и ссылки

### Ссылки на внешние ресурсы
`[Текст ссылки](https://url-адрес.com)`
*Пример:* `[Скачать Blender](https://www.blender.org/download/)`

### Ссылки на облако (Google Drive / Яндекс.Диск)
Используйте этот формат для тяжелых исходников проектов или инсталляторов:
`[📁 Скачать заготовку проекта (Google Drive)](https://drive.google.com/...)`

### Ссылки на локальные файлы в репозитории
Если вы загрузили PDF или архив прямо в папку `files/` вашего GitHub репозитория:
`[📄 Скачать методичку (PDF)](files/methodology.pdf)`

## 4. Как опубликовать материал на сайте

Создание `.md` файла — это 50% работы. Чтобы сайт "увидел" статью, её нужно прописать в манифесте.

### Добавление Урока (в раздел Треки)
1. Откройте `articles/tracks.json`.
2. Найдите массив `lessons` внутри нужного трека.
3. Добавьте объект:
   `{ "title": "Название вашего урока", "file": "название_файла.md" }`
   
**Внимание:** Не забывайте ставить запятую между объектами в JSON!

> [Место для картинки: Скриншот кода JSON в редакторе GitHub, где показано добавление новой строки в список уроков с правильной запятой]

### Добавление Шпаргалки
1. Откройте `articles/cheats.json`.
2. Добавьте новую запись в список:
   `{ "title": "Название шпаргалки", "file": "название_файла.md" }`

## 💡 Чек-лист проверки (Перед "Commit")

1. **Названия**: Файлы и папки названы латиницей (без пробелов, через `_`).
2. **Пути**: Проверьте, что в ссылках на картинки указана верная папка.
3. **Закрытие тегов**: Все блоки кода с апострофами и жирный текст закрыты.
4. **JSON**: Манифест-файлы сохранены корректно, нет лишних запятых в конце списков.
