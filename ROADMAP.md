# Roadmap: Техническое совершенствование и стабильность

Этот раздел посвящен улучшению качества кода, расширению тестового покрытия и архитектурному рефакторингу проекта.

## 🧪 Расширение тестирования (Missing Tests)
*Цель: Покрыть тестами критические узлы и обеспечить отказоустойчивость.*

- [ ] **Тестирование `js/markdown.js` (Custom Tags)**
  - Написать Unit-тесты для регулярных выражений: `[gallery]`, `[compare]`, `[video]`, `[blueprint]`.
  - Проверить корректность обработки путей и параметров внутри тегов.
  - Тестирование механизма сворачиваемых заголовков (`makeHeadersCollapsible`).
- [ ] **Тестирование поиска (`js/search.js`)**
  - Написать тесты для Fuse.js: корректность выдачи результатов при частичном совпадении.
  - Проверить обработку пустых запросов и очистку HTML-тегов в результатах.
- [ ] **Тестирование прогресса и данных (`js/progress.js` & `js/api.js`)**
  - Проверка `localStorage`: поведение при поврежденных данных или старом формате.
  - Имитация сетевых сбоев: как сайт ведет себя при 404 или 500 ошибках на манифестах.
  - Проверка целостности: тест на соответствие файлов в `articles/` и записей в `search_index.json`.
- [ ] **Визуальное и регрессионное тестирование**
  - Внедрить Snapshot-тестирование для проверки верстки (чтобы стили не "плыли" на разных устройствах).
  - Проверка контрастности (A11y) и доступности для экранных дикторов.
- [ ] **Интеграционные тесты навигации**
  - Проверить связь `router.js` -> `ui.js`: вызывает ли смена хэша нужный рендер.

## 🎭 Продвинутые E2E сценарии (Future Work)
*Цель: Проверить сложные пользовательские пути, которые часто ломаются.*

- [ ] **Сквозной поиск:** Открытие модалки -> Ввод -> Клик по результату -> Проверка загрузки статьи.
- [ ] **Синхронизация прогресса:** Клик «Прочитано» -> Переход на главную -> Проверка обновления галочек/процентов.
- [ ] **Интерактивный Граф:** Открытие графа в статье -> Клик по узлу -> Переход на другую статью.
- [ ] **Глубокие ссылки (Deep Linking):** Загрузка сайта по прямой ссылке `/#article:...` (проверка корректности инициализации).
- [ ] **Стресс-тест навигации:** Быстрое переключение статей в сайдбаре (проверка race conditions).

## 🏗️ Архитектурный рефакторинг (Refactoring)
*Цель: Разделить зоны ответственности и упростить поддержку кода.*

- [ ] **Разделение `js/ui.js`**
  - Вынести логику графа знаний в отдельный модуль `js/graph.js`.
  - Создать подмодули для рендеринга конкретных страниц (Tracks, Portfolio, Cheats).
- [ ] **Оптимизация `js/router.js`**
  - Выделить парсер Frontmatter в отдельную утилиту.
  - Отделить логику загрузки данных от логики манипуляции DOM.
- [ ] **Унификация API слоя (`js/api.js`)**
  - Создать единую функцию-обертку для `fetch` с базовой обработкой ошибок и таймаутами.
- [ ] **Компонентизация шаблонов**
  - Избавиться от огромных строк HTML внутри JS. Использовать маленькие функции-фабрики.
- [ ] **Формализация State Management**
  - Добавить в `js/store.js` механизм отписки (unsubscribe), чтобы избежать утечек памяти.

## ⚙️ Продвинутая автоматизация и Производительность
*Цель: Ускорить билд и сделать данные более надежными.*

- [ ] **Улучшение `scripts/build.js`**
  - Добавить валидацию Markdown-файлов (обязательные поля Frontmatter).
  - Реализовать поиск "сиротских" изображений (файлы в `img/`, на которые нет ссылок).
  - Автоматическое создание WebP-копий для тяжелых изображений.
- [ ] **Оптимизация Service Worker (`sw.js`)**
  - Внедрить автоматическую версионность кэша.
  - Добавить страницу-заглушку "Offline Mode".
- [ ] **SEO & Metadata**
  - Автоматическая генерация `sitemap.xml` на основе манифестов.

## 🧹 Наведение порядка (Code Quality)
*Цель: Приведение кода к единому стандарту и повышение безопасности.*

- [ ] **Внедрение Линтинга и Форматирования**
  - Настроить `ESLint` и `Prettier`.
- [ ] **Безопасность рендеринга**
  - Аудит использования `innerHTML`. Замена на `textContent` в результатах поиска и других уязвимых местах.
- [ ] **Консистентность именования**
  - Привести все ID и классы к единому формату (кебаб-кейс, префиксы `btn-`, `icon-`).
- [ ] **Обработка ошибок (UX)**
  - Добавить визуальные "Empty States" и понятные сообщения об ошибках загрузки.
