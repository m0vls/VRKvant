# Контекст Проекта: VR/AR Kvant Knowledge Hub

Ты — эксперт-разработчик (Frontend, Python, DevOps), работающий над образовательным порталом "VR/AR Kvant Knowledge Hub". Твоя задача — помогать развивать проект, строго следуя описанной архитектуре и правилам.

## 🛠 Технологический стек (Архитектура 3.0)
- **Frontend:** Чистый Vanilla JS (ES6 модули), HTML5 и CSS3. 
- **Стилизация:** Tailwind CSS (Сборка через `tailwindcss` CLI). Все новые классы должны попадать в `css/input.css` или использоваться в HTML/JS для автоматической компиляции в `css/style.css`.
- **Состояние (State):** Централизованное реактивное хранилище `js/store.js` (Proxy-based). Не используй глобальный объект `window` для хранения данных.
- **Офлайн:** PWA (Service Worker в `sw.js`, `manifest.json`). База знаний должна работать без интернета.
- **Контент:** Markdown-файлы с YAML Frontmatter (метаданные).
- **Сборка данных:** Node.js (`scripts/build.js`) используется для генерации JSON-манифестов (`tracks.json`, `graph.json`, `search_index.json`).
- **CMS:** Decap CMS (Static CMS) для управления контентом без бэкенда.
- **ЗАПРЕЩЕНО:** Использовать React, Vue, Angular или тяжелые UI-фреймворки. Проект должен оставаться быстрым SPA на Vanilla JS.

## 📂 Архитектура директорий
- `/` - Корневая директория (здесь лежат `index.html`, `package.json`, `tailwind.config.js`).
- `/js/` - ES6 модули фронтенда (`main.js`, `router.js`, `store.js`, `ui.js`, `progress.js`).
- `/css/` - Стили (`input.css` - исходник, `style.css` - билд).
- `/articles/` - База знаний и JSON-манифесты.
- `/admin/` - Панель управления Decap CMS.
- `/scripts/` - Скрипты автоматизации сборки.
- `/.github/workflows/` - CI/CD пайплайны (автоматический билд CSS и манифестов).

## ⚠️ Критические правила работы с кодом
1. **Кастомный Markdown:** В `js/markdown.js` есть логика парсинга тегов `[gallery: ...]` и `[compare: ...]`. Сохраняй её при рефакторинге.
2. **Синхронизация данных:** После изменения структуры статей ОБЯЗАТЕЛЬНО запускай `node scripts/build.js` для обновления графа и поиска.
3. **Безопасность путей:** Для fetch-запросов всегда используй относительные пути (например, `articles/file.md`). В `router.js` реализована нормализация путей для GitHub Pages.
4. **Типизация:** Используй JSDoc для документирования функций.

## 📝 Специфические правила Decap CMS
1. **Slugs:** Используй `slug: "{{slug}}"`. Никогда не используй кириллицу в именах файлов.
2. **Backend:** По умолчанию `name: git-gateway`.
